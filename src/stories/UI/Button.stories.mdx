Buttons allow users to take actions, and make choices, with a single tap.

Buttons communicate actions that users can take. They are typically placed throughout your UI, in places like:

- Modal windows
- Forms
- Cards
- Toolbars

## Button variants

The Button comes with three main variants: primary (default), secondary and quiet but you can add more as you need.

Button also supports an `isCta` prop which will add a `cta` class to the button. This is useful for when you want to make a button stand out more than the others.

## Primary buttons

Primary buttons are high-emphasis, distinguished by their use of obvious actionable styling like a fill. They contain actions that are primary to your app.

## Secondary buttons

Secondary buttons are medium-emphasis buttons. They contain actions that are important but aren't the primary action in an app.

Secondary buttons are also a lower emphasis alternative to primary buttons, or a higher emphasis alternative to quiet buttons.

## Floating Action Button (FAB)

This is a variantion of IconButton that is available via the `isFab` prop.

A floating action button (FAB) performs the primary, or most common, action on a screen.

A floating action button appears in front of all screen content, typically as a circular shape with an icon in its center.

Only use a FAB if it is the most suitable way to present a screen's primary action. Only one component is recommended per screen to represent the most common action.
